# THE SEOUL PRIVATE - v3.0.0 Master Plan

본 문서는 프로젝트의 전체 구조와 핵심 지침을 정의하며, 모든 개발 및 수정 작업 시 최우선으로 참조해야 하는 마스터 가이드입니다.

## 1. 프로젝트 아키텍처 (Project Structure)
기존의 파편화된 구조를 기능 단위(Feature-based)로 재편하여 확장성과 유지보수 효율을 극대화합니다.

### 폴더 구조 및 역할
- **`src/features/`**: 각 탭 및 핵심 기능별 독립 폴더 (예: `Home`, `Auth`, `Artisans`)
  - 각 폴더 내에 `components`, `hooks`, `services`를 두어 '끼워넣기식(Plug-and-Play)' 수정 가능
- **`src/services/`**: Firebase 연동 및 데이터 API 등 전역 비즈니스 로직
- **`src/contexts/`**: 전역 상태 관리 (인증, 언어, 앱 데이터)
- **`src/theme/`**: 화이트/블랙 테마 및 골드 포인트 전용 스타일 정의
- **`src/components/common/`**: 전역 헤더, 푸터, 로딩 스피너 등 공통 UI

## 2. 핵심 개발 지침 (Core Guidelines)
1. **방어적 코딩 (Defensive Coding)**: 에러 발생 시 화이트 스크린 대신 `ErrorOverlay`를 띄워 텔레그램/QR 상담으로 유도합니다.
2. **전수 한글 주석**: 모든 소스 코드 하단에는 해당 코드의 의도와 동작을 설명하는 한글 주석을 포함합니다.
3. **하이브리드 캐싱**: VVIP의 체감 속도를 위해 로컬 캐시를 우선 로드하고 배경에서 데이터를 동기화합니다.
4. **테마 동기화**: `themeContext`를 통해 앱 전체 테마(화이트/블랙)를 관리하며 로고 배경도 이에 맞춰 자동 전환합니다.

## 4. 관리자 전용 규칙 (Admin Protocol)
- **리뷰/이벤트 고정**: `isAdminFixed: true` 데이터가 있는 항목은 사용자 화면에서 삭제/수정이 불가능합니다.
- **챗봇 에스컬레이션**: 결정 트리 기반 시나리오 종료 후 반드시 텔레그램 연결로 유도합니다.
